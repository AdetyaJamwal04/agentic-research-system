================================================================================
                  AGENTIC RESEARCH SYSTEM — SYSTEM ARCHITECTURE
                              Active Components Only
================================================================================

OVERVIEW
--------
An AI-powered research agent that takes a user query, searches the web, 
retrieves from local documents (uploaded PDFs), extracts claims, validates 
evidence, and produces cited answers across multi-turn conversations.

Supports two frontends:
  • Web UI at http://localhost:8000  (chat interface with sessions)
  • CLI via `python main.py`

2. PIPELINE — pipelines/research_pipeline.py
================================================================================

run_research(query, max_iterations=3, use_memory=True, stream_callback=None,
             conversation_context="", local_only=False) → dict

Executes 12 pipeline steps in sequence. Each step is wrapped in try/except
with graceful degradation. Step timings are tracked and streamed.

  Step 0:  Memory Load
           → EvidencePool() — fresh per query
           → ResearchHistory() — SQLite history
           → LongTermMemory.retrieve_relevant(query) — ChromaDB semantic search

  Step 1:  Query Analysis
           → analyse_query(query) → {refined_query, intent, entities, ...}

  Step 1.5: Premise Validation
           → validate_premise(analysis) → {premise_status, correction, ...}
           If premise is false, overrides `effective_query` for rest of pipeline

  Step 2:  Planning
           → create_plan(analysis) → {tasks: [{task_id, description}, ...]}

  Step 3:  Query Generation
           → generate_queries(plan) → {task_queries: [{task_id, queries}, ...]}

  Step 4:  Retrieval
           → retrieve_documents(queries, local_only) → list of tagged docs
           Uses web_search (Tavily) + hybrid_search (ChromaDB + optional BM25)
           If local_only=True: skips web search, retrieves 5 local docs

  Step 5:  Domain Filter
           → filter_documents(query, docs) → filtered list

  Step 6:  Evidence Pool
           → pool.add(docs) — deduplicates and stores

  Step 7:  Reranking
           → rerank_documents(query, docs) → top-K scored docs

  Step 8:  Context Compression
           → compress_context(query, docs) → compressed strings

  Step 9:  Claim Extraction
           → extract_claims(query, evidence) → [{claim, confidence, ...}]

  Step 10: Evidence Synthesis
           → synthesize_evidence(query, evidence) → {evidence_brief, ...}

  Step 11: Reasoning + Critic Loop (up to max_iterations)
           → generate_answer(query, evidence, conversation_context) → string
           → critique_answer(query, answer) → {verdict, score, missing_aspects}
           If insufficient → dynamic_planner revises plan, retrieves more

  Step 12: Persist
           → research_history.log_query(...)
           → ltm.save_verified_claims(...) — quality-gated (critic approved only)


run_research_stream(query, max_iterations, use_memory, conversation_context)
  → async generator, runs run_research in a thread, yields events via queue

3. AGENTS — agents/
================================================================================

Each agent is a stateless function that calls the LLM and returns structured
output. All use `utils.llm_client` for Groq→Gemini failover.

┌─────────────────────────┬──────────────────────────────────────────────────────┐
│ File                    │ Function & Role                                      │
├─────────────────────────┼──────────────────────────────────────────────────────┤
│ query_analyzer.py       │ analyse_query(query) → dict                         │
│                         │ Extracts intent, entities, sub-questions,            │
│                         │ refined query. Returns JSON via get_json_response.  │
│                         │                                                      │
│ premise_validator.py    │ validate_premise(analysis) → dict                   │
│                         │ Checks if the query's premise is factually valid.   │
│                         │ Returns {premise_status, correction, corrected_q}.  │
│                         │                                                      │
│ planner.py              │ create_plan(analysis) → dict                        │
│                         │ Generates a multi-task research plan.               │
│                         │ Returns {tasks: [{task_id, description}]}.          │
│                         │                                                      │
│ query_generator.py      │ generate_queries(plan) → dict                       │
│                         │ Creates specific search queries per task.           │
│                         │ Returns {task_queries: [{task_id, queries}]}.       │
│                         │                                                      │
│ retriever_agent.py      │ retrieve_documents(plan_queries, local_only) → list │
│                         │ Runs web search (Tavily) + hybrid local search.     │
│                         │ If local_only=True: skips web, retrieves 5 local.   │
│                         │ Tags each doc with source_type, url, title.         │
│                         │ Handles async→sync bridging with thread pool.        │
│                         │                                                      │
│ relevance_filter.py     │ filter_documents(query, docs) → list                │
│                         │ LLM-powered domain filter. Removes off-topic docs.  │
│                         │                                                      │
│ evidence_agent.py       │ synthesize_evidence(query, evidence) → dict          │
│                         │ Identifies themes, conflicts, gaps in evidence.      │
│                         │ format_synthesis_for_reasoning(synth) → string.      │
│                         │                                                      │
│ reasoning_agent.py      │ generate_answer(query, evidence,                    │
│                         │                 conversation_context="") → string   │
│                         │ Generates cited answer from evidence.               │
│                         │ Accepts prior Q&A context for multi-turn support.   │
│                         │ Routes through llm_client (Groq → Gemini failover). │
│                         │                                                      │
│ critic_agent.py         │ critique_answer(query, answer) → dict               │
│                         │ Scores answer quality (1-10). Returns verdict        │
│                         │ ("sufficient"/"insufficient"), missing_aspects.     │
│                         │ Drives the retry loop.                               │
└─────────────────────────┴──────────────────────────────────────────────────────┘

│                         │                                                      │
│ hybrid_retriever.py     │ hybrid_search(query, k) → list                     │
│                         │ Combines dense + sparse with Reciprocal Rank Fusion.│
│                         │ Falls back to vector-only if BM25 index absent.     │
│                         │ Returns empty if no ChromaDB index exists.          │
│                         │                                                      │

│                         │ LLM-based reranking. Scores each doc, returns top-K.│
│                         │ Score-aware: skips docs below quality threshold.    │
│                         │                                                      │
│ context_compressor.py   │ compress_context(query, docs) → list                │

│                         │ Routes through llm_client for failover.             │
│                         │                                                      │
│ web_retriever.py        │ (Thin wrapper, imported by retriever_agent)          │

9. FRONTEND — templates/index.html
================================================================================

Single-page chat application served at GET /.

  Components:
    • Session sidebar      — create, rename, delete, switch sessions
    • Chat message area    — user/assistant/system message bubbles
    • Streaming indicator  — live pipeline steps with animated dots
    • Expandable panels    — sources (linked), claims (color-coded confidence)
    • Pipeline steps       — green/red dots showing step completion
    • Input bar            — textarea + 📎 upload + 📄 My Docs toggle + send
    • My Docs toggle       — activates local_only mode (no web search)
    • Toast notifications  — non-blocking feedback for uploads/errors

  Session lifecycle:
    1. Page load → GET /api/sessions → render sidebar
    2. "+ New Research" → POST /api/sessions → empty chat view
    3. User types query → WebSocket /ws/research with session_id
    4. Server streams {type: "step"} events → live pipeline display
    5. Server sends {type: "complete"} → render full answer + metadata
    6. First message → auto-title session
    7. Sidebar refreshes to show updated title
